Arguably, the most important aspect of choosing a programming environment is consistency. The best environment is the one that everybody knows how to set up and use, so it is practical for all members of a team to use the same one.

This website is written with Java in mind for two reasons: most other online resources are also written for Java, and most FTC students appreciate the additional resources and capabilities that it provides. It should not present a scary learning curve to students in the FTC age range.


  • Blocks
    • Runs directly on the Robot Controller - use the REV Hardware Client with USB (REV Control Hubs only) or connect a device to the robot controller's Wi-Fi network and navigate to:
      • Control Hub: 192.168.43.1:8080
      • Phone: 192.168.49.1:8080
      A side-effect of this is that the robot controller must be powered on and nearby in order to edit code.
    • Programs are assembled with visual blocks that are dragged in from a sidebar and connected in chains. Chains are run from top to bottom.
    • There is also a horizontal component. Some blocks produce values and have a tail on the left. They are accepted by blocks that can accept those values.
    • All the features available to your robot (it hides features that your robot lacks hardware for) are displayed and easily accessible in the sidebar. You don't have to discover anything that you can't immediately see there.
    • Other Blockly languages exist. Some have more features than FTC blocks and could be better for an introduction to programming in general. Good Blockly languages include MIT AppInventor and Berkeley's Snap!. All of these languages map to typed code fairly well. (Typed code often contains more concise methods to do some functions, but code can almost always be structured to look like blocks. This is often preferred, as it makes the code easier to read and understand.)
  • The robot controller has a built-in Java editor, usually stylized as OnBotJava.
    • Runs directly on the Robot Controller. See the point about this in Blocks above for further information.
    • Programs consist of typed lines of code, which usually run from top to bottom.
    • Depending on which Blocks language you want to compare to, Java either adds the entire concept of methods, makes them more flexible, or doesn't add much at all (in the case of Snap!). Methods are blocks of code that can be called by name and re-used in multiple places without typing all that code again. For example: public void doSomething() { telemetry.addLine("I'm definitely doing something!"); telemetry.addLine("This function does so much!"); telemetry.update(); } // We've just created a method! (many languages call them "functions", but Java calls them "methods") if (gamepad1.a) { // if [A] on the first gamepad (assign with [Start] + [A]) is pressed doSomething(); // "call" our method } if (gamepad2.a) { // if [B] on the second gamepad (assign with [Start] + [B] is pressed doSomething(); // do the same thing }
